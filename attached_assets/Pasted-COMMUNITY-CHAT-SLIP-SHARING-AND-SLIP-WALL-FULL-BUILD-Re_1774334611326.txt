COMMUNITY CHAT, SLIP SHARING, AND SLIP WALL — FULL BUILD
Read every word before you touch anything. This covers three connected features.

PART 1 — CHAT AVAILABILITY AND TOGGLE
Chat is not always open. It is event driven. Add a Chat Management tab to the admin dashboard.
Inside that tab there is a manual ON/OFF toggle. When admin flips it ON chat goes live and users can see and interact with it. When admin flips it OFF chat is closed and users see a message that says chat opens during live events.
Default behavior for every event:

Admin opens chat when the event goes live
Chat stays open for 24 hours after the event ends
Admin closes it after that 24 hour window
Once we have traffic data we will set an auto close time — for now it is manual

Critical — store the chat on/off state in a database config record. Not hardcoded anywhere. Later we are connecting chat open and close times directly to the events system so it triggers automatically. Build it so that connection can be made without rewriting the chat system. The toggle just updates a value in the database. The chat reads that value to decide if it is open or closed.

Contender — free users:

Cannot submit or share slips
Cannot access expanded emoji library

Challenger — paid subscribers:

Can read and type

Free users cannot see the slip button. The expanded emoji library is locked. They can participate in conversation but premium features are clearly visible as locked to encourage upgrade.

PART 3 — SLIP SYSTEM — CHALLENGER ONLY
Profile side — My Slips tab:
Add a My Slips tab inside the user profile section. Challenger subscribers only can see and access this tab.
User uploads a betting slip image here. Rules:

Accepted formats: JPG, PNG, WebP only
Maximum file size: 5MB per image
When they upload show this notice clearly: "Your slip will be available for 7 days and then permanently deleted. Bragging rights have an expiry."

The slip sits in pending status after upload. Not available to post in chat until admin approves it.
After 7 days from upload date the slip is automatically and permanently deleted from storage and the database. Applies to all slips regardless of status. No exceptions. No extensions.
The My Slips tab shows the user:

Pending slips waiting for review
Approved slips available to post
Days remaining before each slip expires
Clear indicator when a slip has been rejected

Admin moderation queue:
Every uploaded slip goes to the moderation queue inside the admin Chat Management tab. Admin sees:

The slip image
Username who uploaded it
Upload timestamp
Days remaining before auto expiry

Admin actions:

Approve — slip moves to user's approved library, becomes available to post in chat. Admin can also mark a slip as FEATURED — this sends it to the Slip Wall hall of fame.
Reject — slip is permanently deleted, user receives a private in-app notification saying their slip was not approved. No reason given.

Admin can manually delete any slip at any time.
Chat side — posting a slip:
Challenger users see a slip button in the chat input bar. Free users do not see this button.
When a Challenger user clicks the slip button:

Their approved slip library opens as a picker
They select which approved slip to share
It posts to live chat as a message with their username, their badge if they have one, and a SLIP badge on the message
Slip image displays inline in chat

Spam control:
User can only post one slip every 30 minutes in chat. If they try before cooldown expires show them: "You can share another slip in X minutes." Cooldown is configurable by admin in Chat Management config. Default 30 minutes.
Auto cleanup:
Run a daily scheduled cleanup job through the existing cron runner that finds all slips older than 7 days and permanently deletes them from storage and database. Runs every day at midnight.

PART 4 — SLIP WALL — HALL OF FAME
This is a public showcase of the biggest and most impressive approved slips on the platform. Two placements:
On the landing page:
Add a section called something like BIG HITS or THE WALL between the leaderboard section and the pricing section. Shows the most recent featured slips. Each slip card shows:

The slip image
Username of who hit it
A short label if admin adds one — example: 450x parlay, UFC 299
Date it was posted

This is live social proof. New visitors see real people hitting real slips and want to be part of it. This drives subscriptions because you need to be a Challenger to submit a slip.
Maximum 6 slips displayed on the landing page at any time. Admin controls which ones are featured. Newest featured slips push older ones off.
Inside the app:
Add a Slip Wall tab or section inside the community area. Same concept but shows more — full scrollable history of all featured slips going back to launch. Users can browse it anytime even when chat is closed.
How a slip gets on the wall:
Admin reviews a slip in the moderation queue. If it is impressive enough admin hits Approve and Featured. It goes to the user's approved library AND gets added to the Slip Wall. Regular approved slips only go to the user's library. Only featured slips make the wall.
Admin can remove any slip from the wall at any time. Admin can also add a caption or label to a featured slip — example: "450x parlay — UFC 299 — NightHawk"
