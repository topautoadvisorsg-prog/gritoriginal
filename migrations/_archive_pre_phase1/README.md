# Pre-Phase-1 Migration Archive

These migrations were generated by `drizzle-kit` against a **broken config** that only saw `shared/schema.ts` and missed `shared/models/auth.ts`. They were applied to production via raw `npm run db:push --force`, NOT through the migration runner, so:

- The `_journal.json` only registered 2 of 9 files
- A duplicate `0001_*` collision was created
- Production drifted from code in ~16 tables (slips, groups, fight_notes, user_keys, etc. existed in code but not in the DB)

**Archived on 2026-05-21** as part of Week 0 cleanup. The Drizzle config now points at both schema files, and a fresh baseline was generated against the unified schema.

Kept here for historical reference only. Do not apply.
